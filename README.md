# PDF・Word・PowerPoint・画像 結合 / ページ編集ツール

PDF・Word（.docx）・PowerPoint（.pptx）・画像（PNG/JPEG など）をまとめて読み込み、
ページを **並べ替え・回転・複製・削除** して 1 つの PDF に書き出す、ブラウザ完結のツールです。

**ファイルは一切外部に送信されません。** すべてあなたの端末の中（ブラウザ内）だけで処理します。
機密文書も安全に扱えます。

---

## 使い方（3通り）

### 1. ブラウザでそのまま使う（インストール不要）

GitHub Pages 上の公開ページを開くだけで使えます。
ページの表示に GitHub のサーバーを使うだけで、**読み込んだファイルは外部に送信されません**（下記「機密・プライバシー設計」参照）。

### 2. ダウンロードして使う（完全オフライン）

このリポジトリを「Code → Download ZIP」でダウンロード（または `git clone`）して展開し、
下の「起動方法」の手順でローカル起動します。ネット接続なしで動きます。

### 3. 自分のサーバーに置いて使う

フォルダごと任意の静的サーバー（レンタルサーバー等）にアップロードするだけで動きます。
ビルド不要・データベース不要です。

## 起動方法（ローカル）

### かんたん（おすすめ）
`PDF編集ツールを起動.command` を **ダブルクリック** します。
ローカルサーバーが立ち上がり、ブラウザが自動で開きます。

> 初回だけ、macOS が「開発元を確認できない」と警告することがあります。
> その場合は `PDF編集ツールを起動.command` を **右クリック →「開く」** を選び、
> 確認ダイアログで「開く」を押してください（次回からはダブルクリックで起動します）。

使い終わったら、黒いターミナルウィンドウで **Control + C**、またはウィンドウを閉じれば停止します。

### 手動で起動する場合
ターミナルでこのフォルダに移動して、次を実行：

```
python3 -m http.server 8800
```

ブラウザで http://localhost:8800/ を開きます。

> ※ `index.html` を直接ダブルクリック（file://）でも開けますが、ブラウザによっては
> サムネイル表示が動かないことがあります。上記の起動方法を推奨します。

## 使い方

1. **追加** — 「ファイルを追加」、またはウィンドウへ **ドラッグ＆ドロップ**。複数まとめてOK。PDF・Word・PowerPoint・画像を混在できます。
2. **並べ替え** — ページのカードを **ドラッグ** して順番を入れ替え。
3. **回転** — 各カードの ↺ / ↻、または複数選択して上部の「左回転 / 右回転」。
4. **複製 / 削除** — 各カードの ⧉ / 🗑、または選択して一括。
5. **選択** — カード左上のチェック。「全選択」でまとめて操作。
6. **書き出し** — 「PDFを書き出す」。並び順・回転を反映した 1 つの PDF が `結合PDF_日時.pdf` としてダウンロードされます。

各カードの右下には、**元のファイルの何ページ目か**（例 `p.3`）が表示されます。
元のファイルは一切変更されません（書き出しは新しい PDF を作るだけです）。

## Word（.docx）について

Word は **簡易変換** です。文字・見出し・段落・簡単な表は再現しますが、
**複雑な表組み・画像の配置・ヘッダー/フッター・段組みは崩れることがあります。**

- レイアウトを正確に残したい Word は、**Word やプレビューで PDF 保存してから** 読み込むのが最も確実です。
- 変換は「文書を画像にしてページへ焼く」方式のため、変換後 PDF 内の文字は **検索・選択できません**。
- 対応は **.docx** のみ（古い .doc は非対応）。

---

## PowerPoint（.pptx）について

PowerPoint も **簡易変換** です。1 スライド＝ 1 ページとして、スライドを画像化して PDF に焼きます。
文字・図形・画像・スライドの縦横比（16:9 など）は再現しますが、次の点に注意してください。

- **アニメーション・画面切替効果は再現されません**（各スライドの最終的な見た目を 1 枚に描画します）。
- **グラフ（チャート）は崩れる場合があります**。表組みや凝ったレイアウトもずれることがあります。
- **音声・動画は PDF に含めず、自動で除外します。** 読み込み時に「音声 N 件を除外」とお知らせします
  （確認ダイアログは出ません。元の .pptx ファイルは一切変更しません）。
- **隠しスライドもページとして含まれます。** 読み込み時に件数をお知らせするので、不要ならそのページの
  カードを削除してから書き出してください（うっかり混入を防ぐための通知です）。
- Word 同様、変換後 PDF 内の文字は **検索・選択できません**。
- 対応は **.pptx** のみ（古い **.ppt は非対応**）。
- レイアウトを正確に残したい場合は、**PowerPoint で PDF 保存してから** 読み込むのが最も確実です。

## 画像（.png .jpg .jpeg .webp .gif .bmp .svg）について

- **1 画像 = 1 ページ** として PDF 化します。縦横比はそのまま、長辺が A4 の長辺に合うページになります。
- スマホ写真の **EXIF 回転情報を反映** します（横向きに保存された写真も正しい向きで入ります）。
- 長辺 4000px を超える巨大画像は、PDF の肥大とメモリ消費を抑えるため **長辺 4000px まで自動縮小** します。
- 透明部分のある画像（透過 PNG など）は透明を保ったまま、それ以外は白地の JPEG として軽量に埋め込みます。
- アニメーション GIF は **1 コマ目だけ** がページになります。

---

## パスワード付き PDF について

- パスワードで保護された PDF は、読み込み時にパスワードを尋ねます。入力すれば **表示・並べ替え・確認** ができます。
- ただし **書き出し（結合）は、事前にパスワードを解除した PDF** を使うことを推奨します
  （暗号化されたままだと正しく結合できない場合があります）。

## 機密・プライバシー設計

- **外部送信なし** — `index.html` に CSP（Content-Security-Policy）を設定し、`connect-src 'self'` で
  外部へのネットワーク送信を遮断しています。読み込んだ文書がネットに出ることはありません。
  GitHub Pages 版でも同じで、GitHub のサーバーから届くのはページ本体（HTML/JS）だけ。
  あなたが読み込んだ文書・画像がブラウザの外へ出ることはありません。
- **CDN 不使用** — 必要なライブラリはすべて `lib/` にローカル同梱。来歴は `SOURCES.md` に記録（バージョン・入手元・SHA-256）。
- **軽量設計** — 初回に読み込むのは PDF 編集に必須のライブラリ（約 0.9MB）のみ。
  Word・PowerPoint 用の変換ライブラリ（約 2.4MB）は、該当ファイルを追加した時に初めて読み込みます。
- **保存しない** — 文書は処理中のメモリ上だけにあり、ディスクや localStorage 等には保存しません。
  「全消去」でメモリを解放します。ページを離れると内容は消えます。
- **拡張機能に注意** — ブラウザ拡張機能はページ内容を読める場合があります。機密文書を扱うときは、
  拡張機能の少ない／専用プロファイルのブラウザを推奨します。

## フォルダ構成

```
pdf-editor/
├─ index.html      画面とスタイル・CSP
├─ app.js          ロジック
├─ lib/            同梱ライブラリ（CDN不使用）
├─ PDF編集ツールを起動.command   起動スクリプト
├─ sample.docx     動作確認用のサンプル Word
├─ sample.pptx     動作確認用のサンプル PowerPoint（16:9・2スライド・音声1件入り）
├─ README.md       この説明
└─ SOURCES.md      ライブラリの来歴（バージョン・入手元・ハッシュ）
```

| ライブラリ | 役割 |
|---|---|
| pdf-lib | PDF の結合・回転・書き出し |
| pdf.js | ページのサムネイル表示 |
| mammoth | Word(.docx) → HTML |
| html2canvas | HTML → 画像（Word・PowerPoint 変換） |
| Sortable | ドラッグ並べ替え |
| PPTXjs（+ jQuery / JSZip / FileReader.js / divs2slides） | PowerPoint(.pptx) → スライド描画（PDF 変換用） |

## 既知の制限

- Word・PowerPoint は簡易変換（上記）。PowerPoint のアニメーション・切替効果は再現されず、グラフは崩れる場合があります。
- PowerPoint の音声・動画は PDF に含めず自動で除外します（.ppt 旧形式は非対応）。
- パスワード PDF の書き出しは非推奨（上記）。
- 数百MB級・数百ページの巨大 PDF はブラウザが重くなることがあります（合計 100MB 超で確認が出ます）。
- スキャン画像主体の重い PDF や、図形の多い PowerPoint は、変換・サムネイル生成に時間がかかることがあります。
- 画像は再エンコードして埋め込むため、元画像そのままのバイト列ではありません（見た目の劣化はほぼありません）。
- HEIC（iPhone の高効率形式）は非対応です。iPhone 側で JPEG 書き出しにするか、プレビュー等で変換してください。
- 極端に画素数の多い画像（目安：長辺 20,000px 超）は、縮小前のデコード段階でブラウザのメモリを大量に使い、
  タブが落ちることがあります。事前に縮小してから読み込んでください。
